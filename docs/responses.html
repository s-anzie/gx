Core

Responses

Every handler returns a core.Response. Responses are immutable values — you can chain header/status modifiers without mutating the original.

Built-in response builders

BuilderStatusContent-Type
c.JSON(data)200application/json
c.Created(data)201application/json
c.NoContent()204—
c.Text(format, args…)200text/plain
c.HTML(html)200text/html
c.XML(data)200application/xml
c.File(path)200auto-detected
c.Stream(ct, reader)200custom
c.Redirect(url)302—

Chaining modifiers

Use core.WithStatus, core.WithHeader, and core.WithCookie to decorate any response:

response_chain.gogo 
// 201 + custom header + cookie
return core.WithCookie(
    core.WithHeader(
        core.WithStatus(c.JSON(user), 201),
        "X-Request-Id", requestID,
    ),
    core.Cookie{Name: "session", Value: token, HttpOnly: true},
)

Error responses

Use gx.AppError to return structured, client-facing errors. See Error Handling for full details.

error_response.gogo 
var ErrNotFound = gx.E(404, "NOT_FOUND", "Resource not found")

func getUser(c *core.Context) core.Response {
    user, err := findUser(c.Param("id"))
    if err != nil {
        return ErrNotFound.Wrap(err).ToResponse()
    }
    return c.JSON(user)
}
