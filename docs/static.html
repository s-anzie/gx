Features

Static Files

GX provides three methods for serving static assets: directory-based, single-file, and embed.FS. All automatically serve pre-compressed .gz / .br variants when available and the client accepts them.

Directory — app.Static

Maps a URL prefix to a directory on disk. Path traversal is blocked automatically.

static.gogo 
// GET /assets/style.css  →  ./public/style.css
app.Static("/assets", "./public")

// With options
app.Static("/assets", "./public",
    core.StaticMaxAge(24*time.Hour),  // Cache-Control: max-age=86400
    core.StaticIndex("index.html"),    // default index file
    core.StaticBrowse(false),          // disable directory listing
)

Single file — app.StaticFile

static_file.gogo 
app.StaticFile("/favicon.ico",     "./public/favicon.ico")
app.StaticFile("/robots.txt",       "./public/robots.txt")

Embedded FS — app.StaticFS

Serve files embedded at build time using Go's embed package:

static_fs.gogo 
import "embed"

//go:embed public
var publicFS embed.FS

func main() {
    app := gx.New()
    // GET /assets/… maps into the embedded public/ directory
    app.StaticFS("/assets", publicFS,
        core.StaticMaxAge(7*24*time.Hour),
    )
}
