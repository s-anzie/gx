HTTP/3

GX supports HTTP/3 (QUIC) via quic-go. HTTP/3 reduces latency on unreliable networks and supports multiplexed streams without head-of-line blocking.

TLS required. HTTP/3 requires TLS. You must provide a valid certificate and key file, or use the development TLS option.

Starting an HTTP/3 Server

Use gx.WithHTTP3() option and app.Listen() automatically handles both TCP (HTTP/1.1+HTTP/2) and UDP (HTTP/3) protocols with automatic Alt-Svc bootstrap header.

go
app := gx.New(
    gx.WithTLS("cert.pem", "key.pem"),
    gx.WithHTTP3(), // Enable HTTP/3 support
)

// Listen automatically handles HTTP/1.1/2 on TCP and HTTP/3 on UDP
// with automatic Alt-Svc header injection
if err := app.Listen(":443"); err != nil {
    log.Fatal(err)
}

HTTP/1.1 + HTTP/2 Only (Without HTTP/3)

If you don't enable HTTP/3 via gx.WithHTTP3(), Listen() automatically launches just HTTP/1.1 and HTTP/2 on TCP.

go
app := gx.New(
    gx.WithTLS("cert.pem", "key.pem"),
    // No WithHTTP3() option - HTTP/3 is not enabled
)

// Register routes...
app.GET("/", homeHandler)

// Listen only handles HTTP/1.1 + HTTP/2 on TCP
if err := app.Listen(":443"); err != nil {
    log.Fatal(err)
}

HTTP/3 Options

Pass core.H3Option values to Listen() to customize HTTP/3 behavior when HTTP/3 is enabled.

OptionDescription
core.WithoutAltSvcShim()Disable the automatic Alt-Svc header injection that advertises HTTP/3 to HTTP/1.1 and HTTP/2 clients
core.AltSvcMaxAge(d time.Duration)Override how long clients cache the Alt-Svc upgrade hint
go
app.Listen(":443",
    core.AltSvcMaxAge(48 * time.Hour),
)

Development TLS

Use the gx.WithDevTLS() option for local development. It generates a self-signed certificate automatically.

go
app := gx.New(
    gx.WithDevTLS(), // Auto-generate self-signed cert for development
    gx.WithHTTP3(),
)

app.ListenH3(":8443")
Do not use WithDevTLS() in production. The certificate is self-signed and not trusted by real clients.

Notes

  • HTTP/3 uses UDP while HTTP/1.1 and HTTP/2 use TCP. Both can share port 443 simultaneously.
  • When HTTP/3 is enabled, the Alt-Svc shim automatically adds the header Alt-Svc: h3=":443"; ma=86400 (or your custom max-age) to every response, so clients discover the QUIC endpoint.
  • A single app.Listen() call handles all protocol negotiation and server setup automatically. No manual goroutine management needed.
  • GX handlers, middleware, and plugins work identically under HTTP/3 — no code changes required.
