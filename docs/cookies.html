Features

Cookies

GX provides context methods for reading cookies and both an inline setter and a response-chain approach for writing them.

Reading cookies

MethodReturnsDescription
c.Cookie(name)(string, error)Value of named cookie; error if absent
c.CookieDefault(name, def)stringValue or def if absent / error
c.RawCookie(name)(*http.Cookie, error)Full *http.Cookie for all fields
read.gogo 
// Value or error
token, err := c.Cookie("session_token")

// Value with a fallback
lang := c.CookieDefault("lang", "en")

// Full http.Cookie for Path, Domain, Expires …
raw, err := c.RawCookie("pref")

Writing cookies inline

c.SetCookie(name, value) writes the cookie directly onto the response writer using secure defaults (HttpOnly, SameSite=Lax). Use this when you don't need to chain it with response modifiers.

write_inline.gogo 
c.SetCookie("session_token", token)
return c.JSON(http.StatusOK, payload)

Writing cookies via response chain

When building responses functionally, use core.WithCookie to attach a core.Cookie to any response. Full control over every attribute.

write_response.gogo 
import "github.com/s-anzie/gx/core"

resp := core.WithCookie(
    c.JSON(payload),
    core.Cookie{
        Name:     "session_token",
        Value:    token,
        MaxAge:   3600,          // 1 hour
        HttpOnly: true,
        Secure:   true,
        SameSite: http.SameSiteStrictMode,
    },
)
return resp

Clearing cookies

clear.gogo 
return core.WithClearCookie(c.JSON(payload), "session_token")

WithClearCookie sets Max-Age=0 and Expires in the past, instructing the browser to delete the cookie immediately.
