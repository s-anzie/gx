Introduction

GX Framework

GX is a minimal, expressive HTTP framework for Go. It gives you a high-performance radix-tree router, chainable middleware, a typed plugin system, and a rich set of opt-in features — all on top of the standard net/http library.

💡

GX requires Go 1.22+. The module path is github.com/s-anzie/gx.

Design philosophy

  • Thin core, optional everything. The core package is a pure HTTP layer with zero external dependencies. Features are opt-in.
  • Idiomatic Go. Every function signature is a plain Go function. No reflection magic, no global state, no hidden behaviour.
  • Composable plugins. Plugins install middleware in a guaranteed priority order regardless of call order in your code.
  • Zero-alloc hot path. Context objects are pooled via sync.Pool, keeping allocations near zero per request.

Getting Started

Quickstart

From zero to a running API in under a minute.

1

Create a new module

Run these commands in your project directory.

terminal shell 
# Create module
mkdir myapp && cd myapp
go mod init myapp

# Install GX
go get github.com/s-anzie/gx@latest
2

Write main.go

A minimal app with one route and graceful shutdown.

main.go go 
package main

import (
    "github.com/s-anzie/gx"
    "github.com/s-anzie/gx/core"
)

func main() {
    app := gx.New()

    app.GET("/ping", func(c *core.Context) core.Response {
        return c.JSON(gx.Map{"message": "pong"})
    })

    app.Boot(":8080")
}
3

Run it

GX logs startup info and handles SIGTERM / SIGINT automatically.

terminal shell 
# Start the server
go run main.go

# In another terminal
curl http://localhost:8080/ping
# → {"message":"pong"}
✅

GX listens for SIGTERM and SIGINT and performs a graceful shutdown (default 10s timeout). No extra setup required.

Core

Routing

GX uses a radix-tree router for sub-microsecond route resolution. Routes are registered directly on the app, on a group, or on a standalone Router that can be mounted later.

HTTP methods

All standard methods are available as named helpers:

routes.gogo 
app.GET("/users", listUsers)
app.POST("/users", createUser)
app.PUT("/users/:id", replaceUser)
app.PATCH("/users/:id", updateUser)
app.DELETE("/users/:id", deleteUser)
app.HEAD("/users", headUsers)
app.OPTIONS("/users", optionsUsers)

// Register a handler for ALL methods
app.Any("/*path", catchAll)

Route parameters

SyntaxExample URLc.Param("…")
:name/users/42"42"
*name (wildcard)/files/a/b/c"a/b/c"
handler.gogo 
app.GET("/users/:id/posts/:postId", func(c *core.Context) core.Response {
    userID := c.Param("id")
    postID := c.Param("postId")
    return c.JSON(gx.Map{"user": userID, "post": postID})
})

Groups

Groups share a common URL prefix and optional middleware. Groups can be nested — middleware composes in order from outer to inner.

groups.gogo 
// All routes under /api/v1 go through authMiddleware
api := app.Group("/api", authMiddleware)
v1  := api.Group("/v1")

v1.GET("/users",     listUsers)
v1.POST("/users",    createUser)
v1.GET("/users/:id", getUser)

// Nested group — adds adminOnly on top of authMiddleware
admin := v1.Group("/admin", adminOnly)
admin.DELETE("/users/:id", hardDeleteUser)

Standalone Router & Mount

Build a *core.Router independently and mount it at a prefix. Useful for splitting routes across files or packages.

router_mount.gogo 
// In another file / package
func UserRouter() *core.Router {
    r := core.NewRouter()
    r.GET("/users",     listUsers)
    r.POST("/users",    createUser)
    r.GET("/users/:id", getUser)
    return r
}

// In main — mount with optional extra middleware
app.Mount("/api/v1", UserRouter(), authMiddleware)
ℹ️

The router's own Use() middleware runs inside the mount-level middleware. Execution order: mount middleware → router middleware → handler.

Error responses

Use gx.AppError to return structured, client-facing errors. See the Error Handling section for full details.

error_response.gogo 
var ErrNotFound = gx.E(404, "NOT_FOUND", "Resource not found")

func getUser(c *core.Context) core.Response {
    user, err := findUser(c.Param("id"))
    if err != nil {
        return ErrNotFound.Wrap(err).ToResponse()
    }
    return c.JSON(user)
}

Middleware

Middleware is a function with signature func(core.Handler) core.Handler. It wraps a handler, allowing code to run before and after the inner handler.

Writing middleware

middleware.gogo 
func Logger() core.Middleware {
    return func(next core.Handler) core.Handler {
        return func(c *core.Context) core.Response {
            start := time.Now()
            res := next(c)                     // call inner handler
            elapsed := time.Since(start)

            slog.Info("request",
                "method",  c.Method(),
                "path",    c.Path(),
                "status",  res.Status(),
                "elapsed", elapsed,
            )
            return res
        }
    }
}

Registering middleware

ScopeAPIEffect
Globalapp.Use(mw1, mw2)Runs for every request
Groupapp.Group("/api", mw)Runs for routes in this group only
Routeapp.GET("/x", mw, handler)Runs for this single route
Mountapp.Mount("/x", router, mw)Runs before the mounted router's own middleware
register_mw.gogo 
// Global — runs for ALL routes
app.Use(Logger(), Recovery())

// Route-level — only for this handler
app.POST("/upload", RateLimitUpload(), handleUpload)

Execution order

Middleware executes in registration order, outermost first. For a global + group + route chain:

🔗

Global middleware → Group middleware → Route middleware → Handler
Response unwinds in reverse order (like a stack).

Error Handling

GX uses structured AppError values instead of plain Go errors. They carry an HTTP status, a machine-readable code, a human message, optional detail fields, and optional translations.

Defining errors

Define errors as package-level variables so they're reusable and testable:

errors.gogo 
var (
    ErrNotFound     = gx.E(404, "NOT_FOUND",      "Resource not found")
    ErrUnauthorized = gx.E(401, "UNAUTHORIZED",    "Authentication required")
    ErrForbidden    = gx.E(403, "FORBIDDEN",       "Access denied")
    ErrConflict     = gx.E(409, "CONFLICT",        "Resource already exists")
)

Adding context

With(key, value) adds detail fields. Wrap(err) attaches an internal Go error for logging — it is never sent to the client. Both methods return a new AppError (immutable).

error_context.gogo 
return ErrNotFound.
    With("id", userID).
    With("resource", "user").
    Wrap(dbErr).          // logged, not sent
    ToResponse()

The JSON response body sent to the client:

response.jsonjson 
{
    "error": {
        "code":    "NOT_FOUND",
        "message": "Resource not found",
        "details": { "id": "42", "resource": "user" }
    }
}

i18n — translated messages

Pass a gx.Translations map as an optional fourth argument to gx.E(). Call err.MessageFor("fr") to retrieve the localized message.

i18n_errors.gogo 
var ErrNotFound = gx.E(404, "NOT_FOUND", "Resource not found",
    gx.Translations{
        "fr": "Ressource introuvable",
        "es": "Recurso no encontrado",
        "de": "Ressource nicht gefunden",
    },
)

// In a handler — detect language, pick translation
lang := c.AcceptLanguage()          // "fr"
msg  := ErrNotFound.MessageFor(lang)  // "Ressource introuvable"
Features

Validation

GX integrates go-playground/validator/v10 with a one-call API. Validation errors are automatically converted into structured AppError responses with per-field messages.

ValidateOrFail — the happy path

create_user.gogo 
type CreateUserReq struct {
    Name  string `json:"name"  validate:"required,min=2,max=64"`
    Email string `json:"email" validate:"required,email"`
    Age   int    `json:"age"   validate:"gte=18,lte=120"`
}

func createUser(c *core.Context) core.Response {
    var req CreateUserReq
    if err := c.BindJSON(&req); err != nil {
        return gx.ErrBadRequest.Wrap(err).ToResponse()
    }

    // Validates + returns 422 with field errors on failure
    if res, ok := gx.ValidateOrFail(c, &req); !ok {
        return res
    }

    // req is valid ✓
    return c.Created(saveUser(req))
}

Standalone validation

FunctionDescription
gx.Validate(v)Validate a struct — returns nil or ValidationError
gx.ValidateOrFail(c, v)Validate + return 422 Response on failure
gx.ValidateVar(v, "required,email")Validate a single value against tags
gx.RegisterValidator(tag, fn)Register a custom validation function
gx.RegisterValidatorAlias(alias, tags)Create a named alias for a tag combination

Custom validators

custom_validator.gogo 
// Register once at startup
gx.RegisterValidator("slug", func(fl validator.FieldLevel) bool {
    return slugRe.MatchString(fl.Field().String())
})

// Use in struct tags
type Post struct {
    Slug string `validate:"required,slug"`
}
✅

Validation is lazy-initialized via sync.Once — the validator is only created on first use, so startup is unaffected even if you register many custom rules.

File Uploads

GX handles multipart form uploads with magic-byte MIME detection. The detected ContentType is based on file content, not the client-provided header — preventing Content-Type spoofing.

upload.gogo 
func handleUpload(c *core.Context) core.Response {
    file, err := c.FormFile("avatar")
    if err != nil {
        return gx.ErrBadRequest.Wrap(err).ToResponse()
    }

    // Magic-byte MIME detection (JPEG, PNG, GIF, WebP, PDF, ZIP…)
    fmt.Println(file.ContentType)  // e.g. "image/png"
    fmt.Println(file.Size)         // bytes
    fmt.Println(file.Filename)     // original filename

    // Save to disk
    if err := file.Save("./uploads/" + file.Filename); err != nil {
        return gx.ErrInternal.Wrap(err).ToResponse()
    }
    return c.JSON(gx.Map{"saved": file.Filename})
}

// Multiple files under one field
files, _ := c.FormFiles("photos")
for _, f := range files {
    data, _ := f.Bytes()   // read all bytes
    _ = data
}

Content Negotiation

GX parses Accept and Accept-Language headers using full q-factor weighting.

MethodReturnsDescription
c.Negotiate(v)ResponseAuto-select JSON / XML / text based on Accept
c.Accepts("json","xml")stringBest match from candidates (q-weighted)
c.AcceptLanguage()stringPrimary preferred language ("fr", "en"…)
c.AcceptLanguages()[]stringAll languages in preference order
negotiate.gogo 
func getUser(c *core.Context) core.Response {
    user := fetchUser(c.Param("id"))

    // Picks JSON, XML, or text/plain automatically
    return c.Negotiate(user)
}

func translate(c *core.Context) core.Response {
    lang := c.AcceptLanguage()     // "fr"
    msg  := i18n.Get(lang, "welcome")
    return c.JSON(gx.Map{"message": msg})
}

Safe Goroutines

Never use a bare go func() inside a handler — a panic will crash the server. Use gx.Go instead. Panics are recovered, logged with a full stack trace, and the server keeps running.

⚠️

Never capture *core.Context in a goroutine — it is pooled and will be recycled after the handler returns. Extract needed values first.

goroutine.gogo 
func sendWelcome(c *core.Context) core.Response {
    email := c.Param("email") // extract before goroutine

    // Fire-and-forget — panic-safe
    gx.Go(func() {
        mailer.SendWelcome(email)
    })

    return c.NoContent()
}

// With context — honours request cancellation
func processAsync(c *core.Context) core.Response {
    ctx := c.GoContext()
    id  := c.Param("id")

    gx.GoCtx(ctx, func(ctx context.Context) {
        heavyWork(ctx, id)
    })

    return c.JSON(gx.Map{"queued": true})
}

Early Hints (103)

Send 103 Early Hints to push resource hints to the browser before the final response. No-op on HTTP/1.1 connections.

early_hints.gogo 
func page(c *core.Context) core.Response {
    // Sent immediately over HTTP/2 before the response body is ready
    c.EarlyHints(
        core.Preload("/assets/app.css",  "style"),
        core.Preload("/assets/app.js",   "script"),
        core.Preconnect("https://cdn.example.com"),
        core.DNSPrefetch("https://fonts.googleapis.com"),
    )

    // ... generate the page ...
    return c.HTML(html)
}

HTTPS Redirect

Two helpers make HTTP→HTTPS redirection trivial — a middleware for single-server setups and a dedicated redirect server for dual-listen setups.

https_redirect.gogo 
// Option A — middleware: redirects any HTTP request on the same server
app.Use(gx.HTTPSRedirect())

// Option B — dual-listen: HTTP :80 redirects to HTTPS :443
app.ListenHTTPRedirect(":80", ":443")  // runs in background goroutine
app.Boot(":443")                          // main TLS server
🔒

The redirect uses 308 Permanent Redirect (preserves POST bodies) and injects a Strict-Transport-Security header automatically.

Config from Env

gx.LoadConfig[T]() reads environment variables into a typed struct using struct tags. Supports defaults, required fields, and common types.

TagExampleDescription
env:"VAR"env:"DATABASE_URL"Env var name to read
default:"val"default:"8080"Fallback if env var is unset
required:"true"required:"true"Returns error if missing

Supported field types: string, int / int64, float64, bool, time.Duration, []string (comma-separated).

config.gogo 
type Config struct {
    DatabaseURL string        `env:"DATABASE_URL" required:"true"`
    Port        int           `env:"PORT"         default:"8080"`
    Debug       bool          `env:"DEBUG"        default:"false"`
    Timeout     time.Duration `env:"TIMEOUT"      default:"30s"`
    AllowedIPs  []string      `env:"ALLOWED_IPS"`  // comma-separated
}

func main() {
    cfg, err := gx.LoadConfig[Config]()
    if err != nil {
        log.Fatal(err)
    }

    app := gx.New()
    app.Boot(fmt.Sprintf(":%d", cfg.Port))
}
Plugins

Plugin System

Plugins are installed with app.Install() and automatically sorted by priority before the first request. Higher priority runs first.

plugins.gogo 
app.Install(cors.New())
app.Install(compress.New())
app.Install(auth.New(secret))
app.Install(ratelimit.New(...))

// Override execution priority (higher = runs first)
app.Install(myPlugin, gx.PluginPriority(999))

Built-in priority table

Auth750After rate limit
Cache700After auth — cache user-specific responses
Compress100Innermost — compresses final response

Plugin: Auth (JWT)

Validates Authorization: Bearer <token> headers using HMAC-SHA256. Claims are injected into the context store under "claims".

auth_plugin.gogo 
import "github.com/s-anzie/gx/plugins/auth"

app.Install(auth.New(
    auth.WithSecret([]byte(os.Getenv("JWT_SECRET"))),
    auth.WithSkip("/health", "/login"),  // public paths
))

// In a protected handler
claims := auth.ClaimsFrom(c)
fmt.Println(claims.Subject)  // user ID

Plugin: CORS

Handles Origin, preflight OPTIONS requests, and Access-Control-* response headers.

cors_plugin.gogo 
import "github.com/s-anzie/gx/plugins/cors"

// Allow any origin (development)
app.Install(cors.New())

// Restricted (production)
app.Install(cors.New(
    cors.AllowOrigins("https://app.example.com"),
    cors.AllowMethods("GET", "POST", "DELETE"),
    cors.AllowHeaders("Authorization", "Content-Type"),

Plugin: Compress

Compresses responses using Brotli (preferred) or gzip depending on the client's Accept-Encoding. Skips responses below the minimum size.

compress_plugin.gogo 
import "github.com/s-anzie/gx/plugins/compress"

app.Install(compress.New(
    compress.WithLevel(5),       // 1–11 for brotli, 1–9 for gzip
    compress.WithMinSize(1024),  // skip responses smaller than 1 KB
))

Plugin: Cache

Caches GET responses in memory (default) or a pluggable backend. Only 200 responses are cached. Cache keys are based on method + path.

cache_plugin.gogo 
import "github.com/s-anzie/gx/plugins/cache"

app.Install(cache.New(
    cache.WithTTL(60*time.Second),
    cache.WithKeyBuilder(func(c *core.Context) string {
        return c.Method() + ":" + c.Path()
    }),
))

Plugin: Rate Limit

Token-bucket rate limiting. Keyed by IP by default. Supports custom key extractors (e.g. per authenticated user).

ratelimit_plugin.gogo 
import "github.com/s-anzie/gx/plugins/ratelimit"

app.Install(ratelimit.New(
    ratelimit.WithRate(100, time.Minute), // 100 req / minute
    ratelimit.WithKeyFunc(func(c *core.Context) string {
        return c.ClientIP()
    }),
    ratelimit.WithOnExceeded(func(c *core.Context) core.Response {
        return gx.E(429, "RATE_LIMITED", "Slow down").ToResponse()
    }),
))

Plugin: OpenAPI

Auto-generates an OpenAPI 3.1 spec from route Contract declarations. Serves Scalar (default) or Swagger UI documentation.

import "github.com/s-anzie/gx/plugins/openapi"

// Install OpenAPI — serves /docs and /openapi.json

Testing

Testing with gxtest

The gxtest package provides a lightweight test-helper layer that dispatches requests directly to your handlers — no real HTTP server required. Import it with:

go.mod / import
import "github.com/s-anzie/gx/gxtest"

Test a standalone handler

Wrap a single core.Handler function and fire requests at it:

handler_test.go
func TestGetUser(t *testing.T) {
    // Wrap a handler directly
    resp := gxtest.GET(getUser, "/users/42").
        Do(t)

    resp.AssertStatus(t, 200)
    resp.AssertBodyContains(t, "42")
}

Test against a router

Pass your *core.Router (or *gx.App) to test the full routing + middleware stack:

router_test.go
func TestAPI(t *testing.T) {
    app := buildApp()   // returns *gx.App

    resp := gxtest.POST(app, "/users").
        JSON(map[string]any{"name": "Alice"}).
        Header("X-Request-ID", "test-1").
        Do(t)

    resp.AssertStatus(t, 201)
    resp.AssertJSON(t, `{"id":1,"name":"Alice"}`)
}

Transports

HTTP/2 & HTTP/3

GX supports HTTP/2 automatically when you run with TLS, and HTTP/3 (QUIC) via ListenH3.

Enabling TLS (HTTP/2 auto-upgrade)

main.go
app.ListenTLS(":443", "cert.pem", "key.pem")

HTTP/3 (QUIC)

Serving over QUIC requires a valid TLS certificate. Call ListenH3 instead of Listen:

main.go
err := app.ListenH3(":443", "cert.pem", "key.pem")

Detecting the protocol in a handler

handler.go
func info(c *core.Context) error {
    if c.IsHTTP3() {
        // request arrived over QUIC
    }
    return c.JSON(200, map[string]any{
        "proto": c.Request.Proto,
    })
}
Note: HTTP/3 requires that the Alt-Svc header is advertised so browsers upgrade. GX sets this header automatically when ListenH3 is used.
Lifecycle

Lifecycle Hooks

Register callbacks that fire at well-defined points in the application lifecycle — before the server accepts connections (boot), after graceful shutdown (shutdown), and for liveness/readiness probing (health check).

OnBoot

Runs once, just before the server starts accepting connections. Use it to connect to databases, warm caches, or print a startup banner.

main.go
app.OnBoot(func(a *gx.App) error {
    return db.Connect()
})

OnShutdown

Runs after the server has drained active connections. Use it to flush queues, close pools, or clean up resources.

main.go
app.OnShutdown(func(a *gx.App) error {
    db.Close()
    cache.Flush()
    return nil
})

HealthCheck

Register a named health-check probe. GX exposes all registered probes at GET /_health:

main.go
app.HealthCheck("database", func() error {
    return db.Ping()
})

app.HealthCheck("redis", func() error {
    return redis.Ping()
})

A successful probe returns 200 {"status":"ok","checks":{"database":"ok","redis":"ok"}}. Any failing probe returns 503 with the error message.

Graceful shutdown

GX listens for SIGINT / SIGTERM automatically. On signal, it stops accepting new connections, waits for in-flight requests to complete (up to a configurable timeout), then fires all OnShutdown callbacks.

Tip: Override the default shutdown timeout via the GX_SHUTDOWN_TIMEOUT environment variable (e.g. GX_SHUTDOWN_TIMEOUT=15s).
Observability

Observability

GX ships first-class support for distributed tracing, metrics, and structured logging — all wired through standard Go interfaces so you can plug in any backend.

Structured logging

GX uses the standard library's log/slog. Pass your own *slog.Logger at construction time:

main.go
logger := slog.New(slog.NewJSONHandler(os.Stdout, nil))

app := gx.New(gx.Options{
    Logger: logger,
})

Every request is logged automatically with method, path, status, latency, and request_id.

Distributed tracing

Register an OpenTelemetry tracer provider with WithTracing:

main.go
tp := initTracerProvider()  // returns *sdktrace.TracerProvider

app.WithTracing(tp)

A span is created for each incoming request and stored on the context. Retrieve it inside a handler:

handler.go
ctx, span := c.StartSpan("fetch-user")
defer span.End()

user, err := db.GetUser(ctx, id)

Metrics

Pass a Prometheus (or any OpenTelemetry-compatible) meter provider with WithMetrics:

main.go
mp := initMeterProvider()

app.WithMetrics(mp)

// Expose the /metrics scrape endpoint
app.GET("/metrics", gx.PrometheusHandler())
