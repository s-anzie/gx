Middleware

Middleware has the signature func(c *core.Context, next core.Handler) core.Response. It runs before and after the inner handler, with full access to the request and response.

type Middleware = func(c *Context, next Handler) Response

Writing middleware

middleware.gogo 
// Inline middleware function
func Logger(c *core.Context, next core.Handler) core.Response {
    start := time.Now()
    res := next(c)                     // call inner handler
    elapsed := time.Since(start)

    slog.Info("request",
        "method",  c.Method(),
        "path",    c.Path(),
        "status",  res.Status(),
        "elapsed", elapsed,
    )
    return res
}

// Factory pattern — closure over configuration
func RequireRole(role string) core.Middleware {
    return func(c *core.Context, next core.Handler) core.Response {
        claims, ok := gx.TryTyped[UserClaims](c)
        if !ok || claims.Role != role {
            return gx.ErrForbidden.ToResponse()
        }
        return next(c)
    }
}

Registering middleware

ScopeAPIEffect
Globalapp.Use(mw1, mw2)Runs for every request
Groupapp.Group("/api", mw)Runs for routes in this group only
Routeapp.GET("/x", mw, handler)Runs for this single route
Mountapp.Mount("/x", router, mw)Runs before the mounted router's own middleware
register_mw.gogo 
// Global — runs for ALL routes
app.Use(Logger, Recovery)

// Route-level — only for this handler
app.POST("/upload", RateLimitUpload(), handleUpload)

Execution order

Middleware executes in registration order, outermost first.

🔗

Global middleware → Group middleware → Route middleware → Handler
Response unwinds in reverse order (like a stack).
