Plugins

Cache

The cache plugin stores GET responses in memory (or a custom backend) and serves them on subsequent identical requests, reducing handler execution and database load.

terminal
go get github.com/s-anzie/gx/plugins/cache

Basic Usage

go
import (
    "time"
    "github.com/s-anzie/gx"
    "github.com/s-anzie/gx/plugins/cache"
)

app := gx.New()

// In-memory cache, 5-minute TTL (default)
app.Install(cache.New())

// Custom TTL
app.Install(cache.New(
    cache.DefaultTTL(30 * time.Second),
))

Options

OptionDefaultDescription
cache.DefaultTTL(d time.Duration)5mTTL for cached responses
cache.KeyFunc(fn func(*gx.Context) string)SHA-256 of method + path + queryCustom cache key generator
cache.WithBackend(b Backend)in-memoryCustom storage backend (Redis, Memcached, etc.)
cache.OnHit(fn func(*gx.Context, string))nilCallback invoked on cache hit
cache.OnMiss(fn func(*gx.Context, string))nilCallback invoked on cache miss

Custom Cache Key

By default the key is a SHA-256 hash of the HTTP method, path, and raw query string. Override this to factor in user identity, locale, or other request attributes.

go
app.Install(cache.New(
    cache.DefaultTTL(1 * time.Minute),
    cache.KeyFunc(func(c *gx.Context) string {
        // Cache per-user by including the user ID in the key
        userID := c.Header("X-User-ID")
        return c.Path() + ":" + userID
    }),
))

Custom Backend

Implement the cache.Backend interface to use an external store like Redis.

go
// cache.Backend interface
type Backend interface {
    Get(key string) ([]byte, bool, error)
    Set(key string, value []byte, ttl time.Duration) error
    Delete(key string) error
    Clear() error
}

// Example: plug in a Redis backend
app.Install(cache.New(
    cache.DefaultTTL(5 * time.Minute),
    cache.WithBackend(&RedisBackend{client: redisClient}),
))

Cache Headers

The plugin sets an X-Cache response header indicating whether the response was served from cache.

HeaderValue
X-CacheHIT — served from cache
X-CacheMISS — fresh response, now cached

Notes

  • Only GET requests are cached. All other methods are passed through.
  • Only 2xx responses are stored in the cache.
  • The in-memory backend runs a cleanup goroutine every minute to remove expired entries.
  • Install the cache plugin after auth so that authenticated user data is not leaked across users.
