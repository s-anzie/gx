Plugins

CORS

The CORS plugin handles Cross-Origin Resource Sharing headers and responds to preflight OPTIONS requests automatically.

terminal
go get github.com/s-anzie/gx/plugins/cors

Basic Usage

go
import (
    "github.com/s-anzie/gx"
    "github.com/s-anzie/gx/plugins/cors"
)

app := gx.New()

// Use defaults (allow all origins)
app.Install(cors.New(cors.DefaultConfig()))

// Or provide a custom config
app.Install(cors.New(cors.Config{
    Origins:     []string{"https://app.example.com"},
    Methods:     []string{"GET", "POST", "PUT", "DELETE"},
    Headers:     []string{"Authorization", "Content-Type"},
    Credentials: true,
    MaxAge:      12 * time.Hour,
}))

Config Fields

FieldTypeDefaultDescription
Origins[]string["*"]Allowed origins. Use "*" to allow any origin.
Methods[]stringGET, POST, PUT, PATCH, DELETE, OPTIONSAllowed HTTP methods
Headers[]stringOrigin, Content-Type, Accept, AuthorizationAllowed request headers
ExposeHeaders[]stringnilHeaders exposed to the browser
CredentialsboolfalseAllow cookies / authorization headers with credentials
MaxAgetime.Duration12hHow long preflight results may be cached

Default Config

cors.DefaultConfig() returns a permissive configuration suitable for development.

go
cors.DefaultConfig()
// Config{
//     Origins:     []string{"*"},
//     Methods:     []string{"GET","POST","PUT","PATCH","DELETE","OPTIONS"},
//     Headers:     []string{"Origin","Content-Type","Accept","Authorization"},
//     Credentials: false,
//     MaxAge:      12 * time.Hour,
// }

Credentials & Specific Origins

When Credentials: true, the Origins list must contain explicit origins — you cannot use "*".

go
app.Install(cors.New(cors.Config{
    Origins:     []string{"https://app.example.com", "https://admin.example.com"},
    Methods:     []string{"GET", "POST", "PUT", "DELETE", "OPTIONS"},
    Headers:     []string{"Authorization", "Content-Type", "X-Request-ID"},
    ExposeHeaders: []string{"X-Request-ID"},
    Credentials: true,
    MaxAge:      6 * time.Hour,
}))
