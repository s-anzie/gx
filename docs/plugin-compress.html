Compression

The compress plugin transparently compresses responses using Brotli or Gzip based on the client's Accept-Encoding header. Responses below the minimum size threshold are sent uncompressed.

terminal
go get github.com/s-anzie/gx/plugins/compress

Basic Usage

go
import (
    "github.com/s-anzie/gx"
    "github.com/s-anzie/gx/plugins/compress"
)

app := gx.New()

// Use defaults: Brotli preferred, then Gzip; MinSize=1024; Level=-1 (default)
app.Install(compress.New(compress.DefaultConfig()))

Config Fields

FieldTypeDefaultDescription
Algorithms[]Algorithm[Brotli, Gzip]Algorithms to offer, in order of preference
MinSizeint1024Minimum response size in bytes to compress
Levelint-1Compression level. -1 = default, 1–9 for Gzip
Algorithm constantValue
compress.BrotliBrotli (br) — better compression ratio
compress.GzipGzip — wider client support

Custom Configuration

go
app.Install(compress.New(compress.Config{
    Algorithms: []compress.Algorithm{compress.Gzip}, // Gzip only
    MinSize:    512,
    Level:      6, // Balanced speed/ratio for Gzip
}))

Notes

  • The plugin buffers response data until MinSize bytes are accumulated before deciding whether to compress.
  • The Content-Length header is removed from compressed responses since the final size differs.
  • The plugin is installed at priority 100 by default — lower than auth and rate-limiting so it compresses the final output.
  • For static files, consider pre-compressing with .gz/.br variants instead — GX's static file server will serve them directly.
