Features

Validation

GX uses go-playground/validator v10 under the hood. Annotate your structs with validate:"..." tags and call one of the validation helpers.

ValidateOrFail

gx.ValidateOrFail(c, &v) is the most common pattern. It validates the struct, and if invalid returns a ready-made 422 Unprocessable Entity response with field-level errors. It returns (nil, true) on success.

handler.gogo 
type CreateUserReq struct {
    Name  string `json:"name"  validate:"required,min=2,max=50"`
    Email string `json:"email" validate:"required,email"`
    Age   int    `json:"age"   validate:"gte=18,lte=120"`
}

func CreateUser(c *core.Context) core.Response {
    var req CreateUserReq
    if err := c.BindJSON(&req); err != nil {
        return gx.ErrBadRequest.With("detail", err.Error()).ToResponse()
    }

    if res, ok := gx.ValidateOrFail(c, &req); !ok {
        return res // automatic 422 with field errors
    }

    // req is valid — proceed
    return c.JSON(user)
}

Validate

Use gx.Validate(v) when you want to handle the error yourself rather than returning an automatic response.

manual_validate.gogo 
if err := gx.Validate(&req); err != nil {
    return gx.ErrValidation.With("fields", err).ToResponse()
}

ValidateVar

Validate a single value without a struct:

validate_var.gogo 
if err := gx.ValidateVar(email, "required,email"); err != nil {
    return gx.ErrBadRequest.With("email", "invalid email address").ToResponse()
}

Custom validators

Register entirely new tags with gx.RegisterValidator, or compose existing tags into an alias with gx.RegisterValidatorAlias. Both must be called before the first use of Validate.

custom_validator.gogo 
import "github.com/go-playground/validator/v10"

// Custom rule
gx.RegisterValidator("slug", func(fl validator.FieldLevel) bool {
    return slugRegex.MatchString(fl.Field().String())
})

// Alias composing existing tags
gx.RegisterValidatorAlias("password", "required,min=8,max=72")

// Use in struct tags
type Req struct {
    Username string `validate:"required,slug"`
    Password string `validate:"password"`
}
