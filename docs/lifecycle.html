Lifecycle

GX provides boot and shutdown hooks so you can run custom logic when the application starts or stops — for example, connecting to a database, warming caches, or gracefully draining connections.

Boot Hooks

Boot hooks run after all plugins are initialised but before the server starts accepting requests.

go
// type BootHook = func(ctx context.Context) error

app.OnBoot(func(ctx context.Context) error {
    return db.Connect(ctx)
})

app.OnBoot(func(ctx context.Context) error {
    log.Println("server starting...")
    return nil
})

If any boot hook returns an error, the application start is aborted.

Shutdown Hooks

Shutdown hooks run when the server receives a termination signal (SIGINT / SIGTERM) or when ListenWithGracefulShutdown returns. They receive a context with a deadline — respect cancellation so the process exits cleanly.

go
// type ShutdownHook = func(ctx context.Context) error

app.OnShutdown(func(ctx context.Context) error {
    return db.Close()
})

app.OnShutdown(func(ctx context.Context) error {
    return cache.Flush(ctx)
})

app.Boot(ctx)

Boot runs all boot hooks and initialises plugins without starting a TCP listener. This is useful in tests or when you need fine-grained control over startup.

go
ctx := context.Background()
if err := app.Boot(ctx); err != nil {
    log.Fatalf("boot failed: %v", err)
}

// Now safe to use the app as an http.Handler
http.ListenAndServe(":8080", app)

Graceful Shutdown Flow

When ListenWithGracefulShutdown is used, GX handles the full boot-to-shutdown sequence automatically:

  1. Calls app.Boot(ctx) → runs all OnBoot hooks and plugin OnBoot methods.
  2. Starts the HTTP server.
  3. Blocks until SIGINT or SIGTERM.
  4. Calls http.Server.Shutdown(ctx) — drains in-flight requests.
  5. Runs all OnShutdown hooks and plugin OnShutdown methods.
go
app := gx.New()

app.OnBoot(func(ctx context.Context) error {
    return db.Connect(ctx)
})

app.OnShutdown(func(ctx context.Context) error {
    return db.Close()
})

app.GET("/", handler)

// Handles boot, run, and graceful shutdown automatically
app.ListenWithGracefulShutdown(":8080")

Health Endpoints

Register named dependency health checks and expose standard health endpoints.

go
app.Health("database", func(ctx context.Context) error {
    return db.PingContext(ctx)
})

app.Health("redis", func(ctx context.Context) error {
    return redisClient.Ping(ctx).Err()
}, gx.HealthInterval(15*time.Second),
   gx.HealthCritical(true))

// Registers: GET /health, GET /health/live, GET /health/ready
app.RegisterHealthEndpoints()

// Start background health checks (call from OnBoot)
app.OnBoot(func(ctx context.Context) error {
    app.StartHealthChecks(ctx)
    return nil
})
EndpointDescription
GET /healthAll checks + overall status
GET /health/liveLiveness — always 200 if process is running
GET /health/readyReadiness — 200 only if all critical checks pass
