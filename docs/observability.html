Tooling

Observability

GX ships with built-in support for distributed tracing, metrics, and structured logging. Each pillar is configured with gx.New() options and uses no-op implementations by default so you can add observability gradually.

Tracing

Enable OpenTelemetry tracing by passing gx.WithTracing to gx.New().

go
app := gx.New(
    gx.WithTracing("my-service",
        gx.OTELExporter("http://localhost:4318"), // OTLP/HTTP collector endpoint
        gx.TracingSampler(0.1),                   // 10% sample rate
    ),
)

Tracing Options

OptionDescription
gx.OTELExporter(endpoint string)OTLP exporter endpoint (e.g. Jaeger, Tempo, Honeycomb)
gx.TracingSampler(rate float64)Sampling rate 0.0–1.0 (default: 1.0 = 100%)

Metrics

Enable Prometheus metrics by passing gx.WithMetrics to gx.New().

go
app := gx.New(
    gx.WithMetrics(
        gx.PrometheusExporter(":9090"), // Expose /metrics on a separate port
        gx.MetricsBuckets([]float64{0.005, 0.01, 0.025, 0.05, 0.1, 0.5, 1}),
    ),
)

Metrics Options

OptionDescription
gx.PrometheusExporter(addr string)Bind address for the Prometheus /metrics HTTP endpoint
gx.MetricsBuckets(buckets []float64)Custom histogram bucket boundaries for request duration

Automatic HTTP Metrics

When metrics are enabled, GX automatically records the following for every request:

MetricTypeDescription
http_requests_totalCounterTotal requests, labelled by method, route, status, protocol
http_request_duration_secondsHistogramRequest duration in seconds
http_request_size_bytesHistogramRequest body size in bytes
http_response_size_bytesHistogramResponse body size in bytes
http_active_requestsGaugeNumber of in-flight requests

Custom Metrics

go
m := app.Metrics()

// Counter
signups := m.Counter("user_signups_total", "Total user registrations", "plan")

app.POST("/register", func(c *core.Context) core.Response {
    // ... create user
    signups.With("plan", user.Plan).Inc()
    return c.JSON(user)
})

// Histogram
queryDuration := m.Histogram("db_query_duration_seconds", "Database query duration")

// Gauge
activeConn := m.Gauge("db_active_connections", "Active database connections")
activeConn.Set(float64(pool.Stats().InUse))

Structured Logging

Enable structured logging with gx.WithStructuredLogs. GX uses Go's standard log/slog package.

go
import "log/slog"

app := gx.New(
    gx.WithStructuredLogs(
        gx.LogLevel(slog.LevelDebug),
        gx.WithLogFormat(gx.LogJSON),  // text (default) or json
        gx.LogOutput(os.Stderr),
    ),
)

Logging Options

OptionDescription
gx.LogLevel(level slog.Level)Minimum level: slog.LevelDebug, LevelInfo, LevelWarn, LevelError
gx.WithLogFormat(gx.LogText | gx.LogJSON)Output format: LogText (default) or LogJSON
gx.LogOutput(f *os.File)Write destination, e.g. os.Stderr, a log file

Using the Logger in Handlers

go
// Per-request logger (includes request ID, method, path)
app.GET("/users", func(c *core.Context) core.Response {
    c.Log().Info("listing users", "page", c.Query("page"))
    // ...
})

// App-level logger
app.Logger().Info("worker started", "queue", "emails")

All Three Together

go
import "log/slog"

app := gx.New(
    gx.WithTracing("my-api",
        gx.OTELExporter("http://otel-collector:4318"),
        gx.TracingSampler(1.0),
    ),
    gx.WithMetrics(
        gx.PrometheusExporter(":9090"),
    ),
    gx.WithStructuredLogs(
        gx.WithLogFormat(gx.LogJSON),
        gx.LogLevel(slog.LevelInfo),
    ),
)

app.ListenWithGracefulShutdown(":8080")
