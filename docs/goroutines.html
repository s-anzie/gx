Features

Safe Goroutines

Use gx.Go and gx.GoCtx instead of bare go func() calls in handlers. Both provide panic recovery that logs the error and stack trace without crashing the server.

Never capture *core.Context in a goroutine. The context is recycled after the handler returns. Extract all values you need before launching the goroutine.

gx.Go

For fire-and-forget background work:

fire_forget.gogo 
func CreateOrder(c *core.Context) core.Response {
    order := processOrder(c)

    // Extract values before launching the goroutine
    orderID := order.ID
    userEmail := order.Email

    gx.Go(func() {
        sendConfirmationEmail(orderID, userEmail)
    })

    return c.JSON(order)
}

gx.GoCtx

When the background work should be cancelled if the server shuts down, pass the request context to gx.GoCtx:

ctx_goroutine.gogo 
func StartExport(c *core.Context) core.Response {
    exportID := newExportID()
    ctx := c.GoContext()    // safe copy of the context

    gx.GoCtx(ctx, func(ctx context.Context) {
        // ctx is cancelled on server shutdown — honor it
        if err := runExport(ctx, exportID); err != nil {
            markExportFailed(exportID, err)
        }
    })

    return c.JSON(map[string]any{"export_id": exportID})
}

Panic recovery

A panic inside gx.Go or gx.GoCtx is caught, logged with a full stack trace via log/slog, and the server continues serving other requests normally. Without this, a panicking goroutine would crash the entire process.
