Core

Handlers

GX handlers are the core of your application. They receive a Context and return a Response — not the other way around. This makes them pure, testable functions.

Handler Signature

The fundamental design of GX:

go 
type Handler func(c *gx.Context) gx.Response

Handlers receive the entire request in the Context and must return a Response for the client.

Simple Handler

Here's the simplest possible handler:

go 
func hello(c *gx.Context) gx.Response {
    return c.Text("Hello, World!")
}

// Register it
app.GET("/", hello)

Practical Example

A real handler combining multiple concerns:

go 
type CreateUserReq struct {
    Name  string `json:"name"`
    Email string `json:"email"`
}

func createUser(c *gx.Context) gx.Response {
    var req CreateUserReq
    
    // Parse JSON body
    if err := c.BindJSON(&req); err != nil {
        return c.Fail(gx.E(400, "INVALID_JSON", "Body must be valid JSON"))
    }
    
    // Validate
    if req.Name == "" || req.Email == "" {
        return c.Fail(gx.E(400, "MISSING_FIELDS", "name and email required"))
    }
    
    // Business logic (separate function)
    user, err := saveUserToDatabase(req.Name, req.Email)
    if err != nil {
        return c.Fail(gx.E(500, "DB_ERROR", "Could not save user"))
    }
    
    // Return created response
    return c.JSON(user)
}

Best Practices

Keep handlers thin: Handlers should focus on HTTP concerns. Extract business logic into separate functions or packages.

go 
// Good: handler delegates to business layer
func deleteUser(c *gx.Context) gx.Response {
    id := c.Param("id")
    if err := users.Delete(id); err != nil {
        return c.Fail(err)
    }
    return c.NoContent()
}

Use meaningful names: Handler names should describe what they do:

go 
// Good names
app.GET("/users", listUsers)
app.GET("/users/:id", getUserByID)
app.POST("/users", createUser)
app.PUT("/users/:id", updateUser)
app.DELETE("/users/:id", deleteUser)

Handle errors consistently: Use GX's error helpers for uniform error responses:

go 
if err != nil {
    if isNotFoundError(err) {
        return c.Fail(gx.E(404, "NOT_FOUND", "Resource not found"))
    }
    if isValidationError(err) {
        return c.Fail(gx.E(400, "INVALID_DATA", err.Error()))
    }
    return c.Fail(gx.E(500, "INTERNAL_ERROR", "Something went wrong"))
}

Testing Handlers

Because handlers return values, they're trivial to test:

go 
func TestListUsers(t *testing.T) {
    c := gx.NewTestContext("GET", "/users")
    resp := listUsers(c)
    
    if resp.StatusCode != 200 {
        t.Fatalf("expected 200, got %d", resp.StatusCode)
    }
}
